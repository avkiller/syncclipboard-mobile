## 🎯 项目背景

### 当前问题

`src/services/` 文件夹存在以下问题：

1. **职责杂糅**: 一个文件包含多种职责（如 HistoryAPI.ts 包含类型、错误、工具函数等）
2. **重复功能**: 多个文件中存在重复的工具函数
3. **依赖复杂**: 部分服务依赖关系过于复杂
4. **位置不合理**: 部分文件应该放在其他目录

### 重构目标

将 services 文件夹重构为清晰的分层架构：

- **api/** - API 客户端层
- **services/** - 业务服务层
- **storage/** - 存储层
- **utils/** - 工具函数
- **errors/** - 错误处理

### 设计原则

**重要**: 如果一个功能模块只有一个文件，则不创建子目录，直接放在父目录下。

**示例**:

- ✅ `utils/Logger.ts` - 单文件，直接放在 utils 目录
- ✅ `api/AuthService.ts` - 单文件，直接放在 api 目录
- ✅ `api/clients/APIClient.ts` - 多文件模块，创建 clients 子目录

## 🚀 执行步骤

### 第一步：了解当前状态

1. 阅读三份文档，理解重构目标
2. 检查当前目录结构：
   ```bash
   ls -la src/services/
   ```
3. 确认没有未提交的代码更改

### 第二步：选择执行阶段

根据 `REFACTORING_CHECKLIST.md` 选择要执行的阶段：

#### 推荐执行顺序：

1. **阶段 1**: 创建新目录结构（10 分钟，无风险）
2. **阶段 2.1**: 移动存储服务（30 分钟，低风险）
3. **阶段 2.2**: 移动 API 客户端（30 分钟，低风险）
4. **阶段 2.3**: 移动工具服务（20 分钟，低风险）
5. **阶段 3.2**: 拆分错误类（30 分钟，低风险）
6. **阶段 4**: 统一重复函数（25 分钟，低风险）
7. **阶段 3.1**: 拆分 HistoryAPI.ts（1 小时，中等风险）
8. **阶段 5**: 重组业务服务（2-3 小时，高风险）

### 第三步：执行重构

对于每个阶段：

1. **阅读清单**: 仔细阅读 `REFACTORING_CHECKLIST.md` 中对应阶段的任务清单
2. **创建目录**: 如果需要，创建新目录
3. **移动文件**: 使用 IDE 的重构功能移动文件
4. **更新导入**: 使用 IDE 的重构功能更新导入路径
5. **验证代码**: 运行验证命令
6. **测试功能**: 测试相关功能是否正常
7. **记录执行**: 在 `REFACTORING_CHECKLIST.md` 中记录执行情况

### 第四步：验证

每次重构后必须验证：

```bash
# 类型检查
npm run type-check

# 代码检查
npm run lint

# 如果有测试，运行测试
npm run test
```

## ⚠️ 重要注意事项

### 1. 单文件模块原则

**不要为单文件模块创建子目录！**

❌ 错误示例：

```
utils/
├── logger/
│   └── Logger.ts    # 只有一个文件，不需要子目录
```

✅ 正确示例：

```
utils/
└── Logger.ts        # 单文件，直接放在 utils 目录
```

### 2. 使用 IDE 重构功能

**强烈建议使用 IDE 的重构功能**：

- VSCode: 右键文件 → Rename/Move
- WebStorm: Refactor → Move/Rename

IDE 会自动更新所有导入路径，减少错误。

### 3. 分阶段执行

**不要一次性执行所有重构！**

- 每次只执行一个阶段
- 每个阶段完成后验证
- 遇到问题立即停止，解决问题后再继续

### 4. 保留原文件

在移动文件时，可以选择：

- **方案 A**: 保留原文件，添加重导出（推荐）
  ```typescript
  // services/Logger.ts (原文件)
  export * from '../utils/Logger';
  ```
- **方案 B**: 直接删除原文件（需要更新所有导入）

### 5. 测试功能

每次重构后，测试相关功能：

- 移动存储服务 → 测试配置读取、历史记录
- 移动 API 客户端 → 测试服务器连接
- 移动工具服务 → 测试日志、更新检查
- 拆分文件 → 测试相关功能模块

## 🔧 常见问题

### Q1: 移动文件后导入路径错误？

**解决方案**:

1. 使用 IDE 的重构功能（推荐）
2. 全局搜索旧路径，手动替换
3. 运行 `npm run type-check` 查看错误

### Q2: 循环依赖错误？

**解决方案**:

1. 检查是否有相互导入的文件
2. 考虑将共享类型移到 `types/` 目录
3. 使用依赖注入解耦

### Q3: 类型检查失败？

**解决方案**:

1. 检查导入路径是否正确
2. 检查类型定义是否完整
3. 查看 TypeScript 错误信息

### Q4: 功能测试失败？

**解决方案**:

1. 检查文件是否正确移动
2. 检查导入路径是否更新
3. 检查是否有遗漏的文件
